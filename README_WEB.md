# 🌐 Deep Researcher Web界面

## 🚀 快速启动

1. **启动服务器**：
   ```bash
   python web_interface.py
   ```

2. **访问界面**：
   打开浏览器，访问 http://localhost:8080

## 🎯 功能特点

### ✨ 实时研究进度展示
- **进度条显示**：实时显示研究完成百分比
- **步骤追踪**：逐步展示研究过程，包括：
  - 🧠 初步思考分析
  - 🔍 搜索查询生成
  - 📄 论文检索结果
  - 📊 结果分析过程
  - 📝 最终报告生成

### 📱 用户友好的界面
- **响应式设计**：支持桌面和移动设备
- **渐进式展示**：符合阅读习惯的逐步展示
- **实时更新**：使用Server-Sent Events实现实时通信
- **美观动画**：流畅的加载和展示动画

### 🎮 交互功能
- **一键问题**：点击推荐问题快速开始
- **进度跟踪**：实时查看每个研究步骤
- **统计信息**：显示字符数、引用数、阅读时间等
- **学术引用**：高亮显示citation格式引用

## 📋 使用示例

### 1. 输入问题
在输入框中输入您的研究问题，例如：
- "什么是Transformer架构？"
- "深度学习和机器学习的区别"
- "大语言模型的最新进展"

### 2. 观察研究过程
系统会实时显示：
```
🧠 正在进行初步思考和分析...
🔍 生成搜索查询: transformer architecture, attention mechanism...
📄 找到 5 篇论文
📊 正在分析第1轮搜索结果...
🔮 发现需要进一步研究: self-attention details...
```

### 3. 查看最终报告
研究完成后，系统会展示：
- 完整的学术研究报告
- 带引用的详细分析
- 研究统计信息

## 🔧 技术架构

### 后端 (Flask)
- **Flask Web框架**：处理HTTP请求
- **SSE流式通信**：实时推送进度更新
- **线程池**：并发处理多个研究请求
- **会话管理**：独立的研究会话跟踪

### 前端 (HTML/CSS/JavaScript)
- **现代CSS**：渐变背景、动画效果
- **响应式布局**：适配各种屏幕尺寸
- **EventSource API**：接收实时更新
- **动态DOM操作**：流畅的界面更新

### 集成组件
- **DeepResearcher**：核心研究引擎
- **ArXiv API**：学术论文数据源
- **DeepSeek-v3**：大语言模型

## 🔍 研究流程展示

1. **初始化** (5%)
   - 接收用户问题
   - 初始化研究会话

2. **思考分析** (10-20%)
   - LLM进行初步分析
   - 识别关键研究方向

3. **多轮搜索** (30-80%)
   - 生成搜索查询
   - 检索arXiv论文
   - 分析搜索结果
   - 判断是否需要继续

4. **生成报告** (85-95%)
   - 综合所有信息
   - 生成学术报告

5. **完成展示** (100%)
   - 显示最终结果
   - 提供统计数据

## 🛠️ 自定义配置

```python

```

编辑 `templates/index.html` 中的CSS样式

### 添加新功能
在 `web_interface.py` 中添加新的路由处理
